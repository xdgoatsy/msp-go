# 项目待办

本文是仓库唯一的当前待办清单。技术文档和历史记录只引用这里，不再维护独立路线图。

**更新时间**：2026-07-12

## 优先级说明

| 优先级 | 含义 |
|--------|------|
| P0 | 影响数据正确性、安全边界或生产验收 |
| P1 | 影响核心体验、可扩展性或维护效率 |
| P2 | 中长期产品扩展 |

## P0

### AI 学习闭环

- [ ] 实现图片答案 OCR；保持解析成功前不创建 attempt、diagnosis、learning session 或 DKT 更新，并补应用层、HTTP、前端和真实图片验收。
- [ ] 完善通用数学求解能力，覆盖超出当前本地模板和基础答案比较范围的题型，并定义可解释的失败与降级契约。
- [ ] 将 Tutor 当前“单次生成后 SSE 输出”升级为 token 级流式，验证取消、超时、断连和部分输出处理。
- [ ] 对外部 provider 的 Tutor、Portrait、Diagnostician、Math Solver、Question Parser 和 Question Generator 做真实运行质量验收。

### AI 自主练习加固

- [ ] 将每学生每分钟 10 次的进程内生成限流迁移到 Redis 或网关，确保多实例一致。
- [ ] 定义生成题质量复核、保留期限、清理策略和错误题回算流程。
- [ ] 建立服务端可信的 Tutor 题目上下文，降低仅依赖提示词和提交后 solution 权限造成的提前泄题风险。
- [ ] 明确 AI 自主练习对个人 DKT、教师掌握度视图和班级统计的影响边界，并补跨来源统计测试。
- [ ] 在配置真实 `question_generator` 模型的环境完成生成、持久化、作答、诊断和导师跳转的浏览器闭环验收。

### 生产验收

- [ ] 完成 Docker/Compose 实机 smoke，并记录镜像构建、迁移、健康检查和服务重启结果。
- [ ] 完成非 AI API、学生/教师/管理员核心浏览器流程和外部集成验收。
- [ ] 建立性能基线，覆盖登录、题目列表、练习提交、学习会话和教师聚合查询。
- [ ] 对历史纯图片答案数据进行扫描，决定是否清理或回算已受影响的 attempt、DKT state 和 profile mastery。

## P1

### 契约与测试

- [ ] 为当前 Go API 建立可重复的 OpenAPI 导出和契约冻结流程，关闭迁移跟踪中的旧 P0/R1 状态。
- [ ] 补充真实 PostgreSQL Repository 集成测试，重点验证默认值、枚举、级联、事务和迁移兼容性，关闭旧 R2 状态。
- [ ] 复核迁移跟踪文档中的 P0 和已过时 OPEN 风险，按当前 Go-only 架构关闭或重述；保留必要的阶段证据。
- [ ] 补充数据库 Schema 和关键读模型说明，避免依赖超长迁移记录理解当前结构。

### 安全与运维

- [ ] 在生产环境复核注册策略、上传鉴权与限流、教务凭据存储、管理端点网段、TLS 和安全响应头。
- [ ] 持续运行前端依赖审计与 `govulncheck`，升级后记录兼容性测试；历史发现以新的扫描结果为准。
- [ ] 为允许访问私网的 OpenAI-compatible 网关、私有对象存储或校园代理设计显式 allow-list 和审计机制，避免放宽默认出站保护。

2. 后端迁移阶段相关事项还要同步更新 [迁移跟踪文档](backend-python-to-go-refactor.md) 的状态、日期、验证命令、结果、交付物和残余风险。
3. 已完成且需要保留追溯信息的专项方案或报告移入 [历史归档](archive/README.md)。
